## Research Abstract

This repository presents a research-grade intrusion detection workflow for **Vehicular Ad Hoc Networks (VANETs)** built around **Basic Safety Message (BSM)** streams generated by **F2MD/SUMO-based scenarios**. The system is not limited to a single classifier. Instead, it combines:

- BSM-derived kinematic and temporal features
- OBU plausibility and protocol-consistency evidence
- attack-family-specialized RSU detection heads
- probability fusion into a final anomaly score
- adaptive sender trust management for robust final decisions

The resulting design is suitable for both **offline evaluation** and **live monitoring**, allowing the same core methodology to support reproducible experiments, model training, verification, and runtime detection.

## Why This Repository Matters

- It reflects a **thesis-aligned multi-stage IDS pipeline** rather than a toy classifier.
- It separates **feature engineering**, **attack-family reasoning**, **fusion**, and **trust adaptation**.
- It supports both **offline labeled datasets** and **live F2MD-produced BSM directories**.
- It preserves a research workflow that is useful for **evaluation, discussion, replication, and demonstration**.

## System Architecture

```mermaid
flowchart LR
    A[F2MD / SUMO Scenario] --> B[BSM Extraction]
    B --> C[CSV Preparation]
    C --> D[attack_id Injection]
    D --> E[Normalization]
    E --> F[Feature Engineering]
    F --> G[Sybil Features]
    G --> H[OBU Evidence Flags]
    H --> I[RSU Multi-Head Models]
    I --> J[Probability Fusion]
    J --> K[Adaptive Trust Update]
    K --> L[Final Intrusion Decision]
```

## Core Detection Logic

The implemented workflow revolves around five technical layers:

1. **Data normalization**
   Every extracted or raw BSM source is transformed into a stable analytical format before scoring.

2. **Feature engineering**
   The system derives motion, timing, heading, consistency, rate, jitter, and windowed behavioral features from each sender stream.

3. **Specialized attack-family modeling**
   Dedicated heads target distinct behavioral families such as position/speed manipulation, replay/stale patterns, DoS behavior, and Sybil-style identity abuse.

4. **Fusion**
   Head outputs are aggregated into a unified score `p_final`, providing a more defensible final signal than relying on one isolated model.

5. **Trust gating**
   Final alerts are moderated through adaptive trust tracking per sender, which is especially important in VANET settings where false accusation of benign vehicles is costly.

| Family | Attack IDs | Research Meaning |
|---|---|---|
| `pos_speed` | `1..9` | Position and speed falsification behaviors |

## Quick Start

### 1. Extract BSM data into CSV

```bash
python vanet_ids_v2.py extract \
  --extract_root /home/instantf2md/F2MD/f2md-results/LuSTNanoScenario-ITSG5 \
  --extract_out /tmp/features.csv
```

### 2. Inject attack labels

```bash
python vanet_ids_v2.py label \
  --label_input /tmp/features.csv \
  --attack_id 13 \
  --label_output /tmp/features_labeled.csv
```

### 3. Train the thesis-aligned RSU models

```bash
python vanet_ids_v2.py train-rsu \
  --input /tmp/features_labeled.csv \
  --models_dir ./release_v3
```

### 4. Run offline detection

```bash
python vanet_ids_v2.py detect-offline \
  --models_dir ./release_v3 \
  --input /tmp/features_labeled.csv \
  --output /tmp/detect_offline.csv
```

### 5. Verify outputs

```bash
python vanet_ids_v2.py verify \
  --labels /tmp/features_labeled.csv \
  --detect_csv /tmp/detect_offline.csv \
  --outdir ./verify_out
```

## Operational Pipelines

### Offline Evaluation Pipeline

Use this path when you already have extracted CSV data with labels or when you want reproducible experimental evaluation.

1. Extract BSM features from F2MD outputs
2. Add `attack_id`
3. Train RSU multi-head models
4. Detect offline
5. Verify performance by label and attack family

### Realtime Pipeline

Use this path when F2MD is actively generating raw `.bsm` files.

```bash
python vanet_ids_v2.py start-daemon
python vanet_ids_v2.py run-scenario
python vanet_ids_v2.py detect-live \
  --models_dir ./release_v3 \
  --input /home/instantf2md/F2MD/f2md-results/LuSTNanoScenario-ITSG5 \
  --source_kind raw-dir \
  --output ./detect_live.csv
```

## Scientific Notes

### Methodological Characteristics

- The runtime is **not a single-model detector**.
- The system uses **windowed per-sender temporal reasoning**.
- OBU evidence is integrated as a first-class signal rather than post-hoc decoration.
- Sender trust is updated dynamically to stabilize final alerting behavior.

### Why False Positives Matter

In VANET security, a false alarm can lead to the isolation of a legitimate vehicle. For that reason, the trust-gated design is not cosmetic; it is central to the deployment logic of the IDS.
